MetaBoost API Specification

MetaBoost provides a cryptocurrency-agnostic REST API for storing and retrieving Value4Value payment metadata for podcast applications. Store payment data from any cryptocurrency network (Bitcoin Lightning, Monero, or others) using a flexible, unified metadata structure.

Value4Value Model: This API is designed for the podcast Value4Value ecosystem, where listeners support creators directly through cryptocurrency payments. The metadata structure is flexible and payment-type agnostic - use the same schema regardless of which cryptocurrency network processed the payment. Store boost messages, sender information, payment amounts, and episode attribution.

Base URL

https://metaboost.vercel.app

Authentication

Most endpoints require no authentication. Update and delete operations require an updateToken that is returned when creating metadata.

‚ö†Ô∏è Security Note: Store the updateToken securely. Anyone with this token can modify or delete your payment metadata.

Metadata Object (bLIP-0010)

The metadata object follows the bLIP-0010 specification for Value4Value payment metadata. This is a unified, payment-agnostic structure that works the same way for Lightning, Monero, or any other cryptocurrency network. The type field indicates which network was used, but the metadata fields remain consistent across all payment types.

Based on bLIP-0010: This API stores the same metadata fields that are used in Lightning TLV records for Value4Value payments. The structure is cryptocurrency-agnostic and works identically regardless of which payment network processes the transaction. See the bLIP-0010 specification for complete field documentation.

Cryptographic Signatures

Optionally sign payment metadata to prove authenticity. The API stores signatures but does NOT verify them - verification is the responsibility of the receiving application.

‚ö†Ô∏è Critical: JSON field order affects signatures! You MUST canonicalize (sort keys alphabetically) before signing/verifying.

Signing Process

  1. Canonicalize the metadata JSON (sorted keys, no whitespace)
  2. Hash the canonical string (e.g., SHA-256)
  3. Sign the hash with your private key (secp256k1, ed25519, etc.)
  4. Encode signature as hex string

Example: Canonical JSON

// JavaScript canonicalization function
function canonicalizeJSON(obj) {
  if (obj === null) return 'null';
  if (typeof obj !== 'object') return JSON.stringify(obj);
  if (Array.isArray(obj)) {
    return '[' + obj.map(canonicalizeJSON).join(',') + ']';
  }
  
  // Sort keys alphabetically
  const sortedKeys = Object.keys(obj).sort();
  const pairs = sortedKeys.map(key => {
    return '"' + key + '":' + canonicalizeJSON(obj[key]);
  });
  return '{' + pairs.join(',') + '}';
}
